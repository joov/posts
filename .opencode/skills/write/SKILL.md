## What I do

- Read the project's PLAN.md for structure, messaging, and target audience
- Read the project's KNOWLEDGE.md for source material and quotes
- Read any .mmd graph files in the project folder to reference or embed
- Write the full post as a Markdown file named `POST.md` in the project folder
- Follow the plan's structure section by section
- Include at least one anschauliches Beispiel (vivid analogy) as specified in the plan
- Reference graphs at the appropriate points in the text (e.g. "[Graph: Layered Context]")
- Cite sources where appropriate (author name, not full URLs -- URLs go in a references section at the end)
- Write in the author's voice: analytical, strategic, behind-the-scenes perspective
- Language: English (unless the user specifies otherwise)
- Target reading time as specified in the plan's meta section


## Writing style

- Clear, direct, no fluff
- Short paragraphs (2-4 sentences)
- Use subheadings to break up sections
- Bold key terms on first mention
- Use quotes from sources to add authority
- End with a clear takeaway or call to action
- Suitable for Medium publication
- Avoid tables. Emulate them in Mermaid instead (Medium does not support tables)

## When to use me

Used after plan and graphs are created. This is the main content creation step.
